Upgrade my existing weather app into an intelligent micro-climate weather system.

Tech stack:

* Backend: Node.js with Express
* Frontend: existing frontend can remain, but update it if needed
* Database: PostgreSQL
* Machine Learning: Python with scikit-learn
* Environment: Replit

Goal:
Turn the project into a system that:

1. Collects weather data over time for specific locations
2. Stores historical observations in PostgreSQL
3. Makes short-term predictions such as “Will it rain in the next 2 hours?”
4. Compares predictions with actual outcomes
5. Learns and improves over time

Build this using clean, modular, beginner-friendly but production-structured code.

================================

======================

Create PostgreSQL tables for:

* id

* longitude

B. weather_observations

* id
* latitude
* longitude
* observed_at
* temperature
* humidity
* pressure

* cloud_cover

* weather_code
* created_at

C. weather_predictions

* id

================================
2. WEATHER DATA COLLECTION
==========================

Use Open-Meteo API to fetch real weather data for a location.

For each tracked location:

* fetch current weather data
* fetch hourly weather values if needed
* store the observation in weather_observations

Create a service for weather fetching.

================================
3. SCHEDULER / AUTOMATION
=========================

Add an hourly job using node-cron or a similar package.

Every hour:

* get all active tracked locations
* fetch weather data for each location
* store observations in the database

Also allow manual triggering through an API endpoint for testing.

================================
4. MACHINE LEARNING PIPELINE
============================

Add a Python script for training a first machine learning model.

Use scikit-learn.
Start with a simple model such as RandomForestClassifier or LogisticRegression.

Prediction target:

* rain in the next 2 hours (yes/no)

Features can include:

* temperature
* humidity
* pressure
* windspeed
* cloud_cover
* hour of day
* month

The Python script should:

* read historical observation data from PostgreSQL or from an exported CSV
* generate labeled training data
* train the model
* save the trained model to disk

Also create a Python prediction script that:

* receives weather feature values
* loads the saved model
* returns prediction and confidence

================================
5. NODE + PYTHON INTEGRATION
============================

From the Node.js backend:

* call the Python prediction script when making an intelligent forecast
* return prediction results to the frontend

Keep this integration modular and easy to replace later.

================================
6. API ENDPOINTS
================

Create these endpoints:

GET /

* return a simple health message

POST /locations

* add a tracked location
* accepts name, latitude, longitude

GET /locations

* return all tracked locations

POST /collect

* manually collect and store weather data for all active locations

GET /weather/current?lat=...&lon=...

* fetch current weather for given coordinates

GET /predict?lat=...&lon=...

* fetch latest weather data
* run intelligent prediction
* return:

  * current weather
  * prediction text
  * confidence
  * reason/explanation if available

GET /metrics

* return simple system metrics such as:

  * total observations
  * total predictions
  * prediction accuracy if enough data exists

POST /train

* trigger the Python training pipeline manually

This is important because the system must learn from results over time.
